API Authentication

This API utilizes basic authentication. The email and password of a user are used as username and password. As stated on the wikipedia page, these values are not encrypted during transmission and thus HTTPS is required in production. The only resource that does not require authentication is the post to a user.

User

POST /users

Creates a new user
Resource: POST http://cobot.emilflach.com/users

Input

Variables: none
Body: 
        
            'name' => 'required|string|min:4',
            'email' => 'required|email|unique:users',
            'password' => 'required|string|min:6',
            'location' => 'required|integer'

Output

HTTP status code: 201
JSON Response: 
        
            "success": {
            "message": "User successfully created.",
            "status_code": 201
            }

GET /users/{user_id}

Shows information of a specific user.
Resource: GET http://cobot.emilflach.com/users/{user_id}

Input

Variables: int user_id
Body: none

Output

HTTP status code: 200
JSON Response: 
            
                "data": {
                "id": "2",
                "name": "Voluptatum.",
                "location": "A corporis."
                }

PUT or PATCH /users/{user_id}

Update all or some information of a specific user.
Resource: PUT http://cobot.emilflach.com/users/{user_id}

Input

Variables: int user_id
Body: 
        
            'name' => 'string|min:4',
            'email' => 'email|unique:users',
            'password' => 'string|min:6',
            'location' => 'integer'

Output

HTTP status code: 200
JSON Response: 
            
                "success": {
                "message": "User successfully updated.",
                "status_code": 200
                }

DELETE /users/{user_id}

Delete a specific user.
Resource: DELETE http://cobot.emilflach.com/users/{user_id}

Input

Variables: int user_id
Body: none

Output

HTTP status code: 200
JSON Response: 
            
                "success": {
                "message": "User successfully deleted.",
                "status_code": 200
                }
